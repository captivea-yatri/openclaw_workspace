Key Takeaways

  • Critical breaches are the missing Read permissions on Contact, Sales, Accounting, Employee, Goal, Attendance, and Helpdesk.
  • Attendance is completely locked out despite the matrix granting full admin rights.
  • Some modules (Accounting, Attendance) lacked a simple required “name‑like” field, preventing the script from testing Create/Update even though the matrix expects those actions.

Recommendations

  1. Grant the Read permission to the Administrative role (or directly to the user) for Contact, Sales, Accounting, Employee, Goal, Attendance, and Helpdesk.
  2. Give the Administrative role full C R U D rights on the hr.attendance model.
  3. If you want to test Create/Update on Accounting and Attendance, add a required char field (e.g., name) or use an existing required field.
  4. Re‑run the test script after the permission changes to confirm the failures are resolved.
