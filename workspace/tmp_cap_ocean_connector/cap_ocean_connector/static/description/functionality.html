Ocean.io Connector – Wizard Functionalities

1️⃣ People Lookup Wizard

FeatureDescription
PurposeRetrieve the most accurate personal profile for a prospect (name, title, email, phone, LinkedIn, company) from Ocean.io.
Key inputsOcean.io Instance, optional Partner link, Lookup method (Ocean ID, LinkedIn URL/handle, email, or name + company domain).
OptionsApply Overwrite – force replacement of existing Odoo fields with values returned by Ocean.io.
Async revealTick Reveal Emails or Reveal Phones – Ocean.io processes asynchronously and POSTs verified data back to the webhook.
ResultImmediate display of enriched data; “Apply to Contact” writes data back; async reveal updates later via webhook and creates a log entry.
When to useWhen you have only a name or LinkedIn URL and need richer details, or you want to enrich contacts one‑by‑one.

2️⃣ Lead Prospect Wizard

FeatureDescription
PurposeSearch Ocean.io for multiple prospects that match a set of filters (job title, seniority, location, industry, company size, etc.) and import them as Odoo CRM leads.
Key inputsOcean.io Instance, filter fields (job title, location, company domain, keywords, seniority, etc.), max results.
OptionsCreate Leads automatically – each matching prospect becomes a `crm.lead` record.
Async revealNot applicable – results are returned synchronously.
ResultList of matched prospects; select rows and click “Import Selected” to create Odoo leads. A short API log is stored.
When to useBuilding a fresh pipeline of qualified leads based on criteria you define, or quickly populating the CRM after a campaign.

3️⃣ Company Enrichment Wizard

FeatureDescription
PurposePull structured company‑level data (industry, employee count, revenue, tech stack, HQ, website, social profiles) from Ocean.io and enrich the Odoo `res.partner` record.
Key inputsOcean.io Instance, Partner (choose the company) or manually enter a domain/website, optional Ocean Company ID.
OptionsApply Overwrite – replace existing fields on the partner with fresh data.
Async revealNot needed – enrichment API returns the full payload synchronously.
ResultWizard displays the enriched profile; clicking “Apply to Partner” writes new values (industry, employees, revenue, etc.) back to `res.partner`.
When to useWhen you have a company record with only a name or domain and need up‑to‑date firmographic data, or for periodic database clean‑up.

4️⃣ Async Reveal Options (shared)

FeatureDescription
Reveal EmailsRequests verified email addresses; Ocean.io processes asynchronously and POSTs to the webhook when ready.
Reveal PhonesSame flow as emails but for phone numbers.
Why async?Some data requires extra verification and can take seconds to minutes; async handling prevents UI blocking.
VisibilityAll webhook attempts appear in **Ocean Webhook Log** with status `received → processed` or `error`.

5️⃣ Instance Management Form

FeatureDescription
API TokenYour Ocean.io secret token – required for all API calls.
Public HTTPS URLBase URL of your Odoo instance (auto‑filled from `web.base.url` if left blank). Used to build the webhook endpoint.
Webhook URL (computed)`https:///webhooks/ocean?secret=&db=` – copy‑paste into Ocean.io configuration.
Webhook Secret / TokenRandom UUIDs generated on record creation; secret verifies inbound calls, token is for legacy paths.
Credit BalanceButtons **Test Connection** and **Refresh Credits** display remaining recurrent/one‑time credits and daily rate‑limit left.
ActionsTest Connection (validates token), Test Webhook URL (GET ping), Refresh Webhook URL (re‑compute after URL change).
Status`connection_status` shows *unknown / connected / failed* with a short message for troubleshooting.

6️⃣ Webhook Log View

FeatureDescription
Log entriesOne row per inbound POST/GET: payload type, HTTP status, `records_updated`, and a short human‑readable message.
FilteringFilter by `payload_type` (e.g., `reveal_emails`, `enrich_companies`, `ping`).
DebugRaw JSON payload is stored in `payload_json` (truncated to 500 KB) for deep inspection.
Audit trailProvides a full history of what Ocean.io has sent to your Odoo instance – essential for compliance and debugging.
